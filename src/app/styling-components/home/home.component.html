¿Estilos en Angular?

Angular revoluciona la forma en que diseña CSS para aplicaciones web modernas. Pero antes de que puedas convertirte en un ninja de estilo angular, debes dar un paso atrás y considerar las muchas y diferentes maneras en que se pueden hacer las cosas. En este curso, Diseño de aplicaciones angulares, explorará cómo el marco se alinea con los componentes web y lo que eso significa para agregar estilos a componentes y aplicaciones en general. A lo largo de este curso, desarrollará sistemas orientado a la organización, escala y mantenibilidad para HTML y CSS dentro de aplicaciones Angular. A medida que se sienta más cómodo agregando y diseñando componentes dentro de estos sistemas, podrá considerar los temas de los componentes y las formas en que puede transformar su apariencia bajo ciertas circunstancias. Al final de este curso, tendrá un sólido conocimiento de cómo Angular procesa CSS y las diferentes formas en que se puede aprovechar como parte de un diseño general sistema para crear aplicaciones web hermosas, organizadas, mantenibles y preparadas para el futuro.

Estilo de aplicaciones angular

Una inmersión en cómo funcionan los estilos en Angular. Cubrimos cómo las diferentes vistas y los modos de encapsulación funcionan, lo que significan en cuanto al código compilado y el alcance, y por qué es posible que queramos usar cada uno de ellos. Luego pasamos a las diferentes formas en que podemos agregar estilos a los componentes, cómo Angular los maneja y cómo su prioridad fluye a través. Después de una exploración completa de los estilos en Angular, seguiremos adelante. a cómo se hace la emulación de Shadow DOM, los selectores de CSS que se emulan, qué se ven cuando se renderizan en el navegador, y cómo/por qué querríamos usar a ellos.

Enfocado principalmente en el modo de encapsulación de vista "Emulado" predeterminado para Angular componentes. Cubrimos conceptos sobre cómo los estilos globales, variables y mixins pueden ser manejados. Analizamos la implementación de sistemas para selectores y nombres de CSS con convenciones. Luego creamos un sistema que nos permitirá aplicar unidades relativas con previsibilidad. Exploramos la creación de archivos bien organizados y estructurados y abordamos los selectores que usamos, así como los problemas relacionados con las anulaciones de estilo. Nosotros exploramos ideas sobre código, archivo y estructura y organización de directorios. Y finalmente, aprovechamos el siempre popular preprocesador SASS para crear relaciones entre los estilos dentro de componentes individuales o entre múltiples componentes haciéndolos más fáciles de editar y mantener con el tiempo.

Enfocado en crear componentes que tengan múltiples temas y exploramos algunos diferentes maneras en que podemos aplicarlos. Primero establecemos explícitamente el tema por instancia agregando clases y propiedades a los componentes y exploramos como SCSS/CSS se vería. Luego buscamos proporcionar temas conscientes del contexto para componentes que cambiarán automáticamente los estilos en función de su entorno utilizando la pseudo clase de contexto de host. Luego buscamos formas de proporcionar entradas como properties después agregamos lógica para verificar las clases agregadas para aplicar el tema adecuado. También, exploramos formas de alterar condicionalmente el marcado del componente y plantillas cuando sea necesario. Y finalmente, observamos un enfoque completamente diferente de tematización de componentes utilizando propiedades personalizadas de CSS.
